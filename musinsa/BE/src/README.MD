## 카테고리 별 DB 분리 설계 이유

```
장점
1. 카테고리 별로 고유 속성이 존재할거라 생각하고 분리함
   각 카테고리에 맞게 최적화된 스키마와 인덱스를 설계할 수 있다.
    예를 들어, 스포츠 용품 카테고리는 다른 패션 액세서리 카테고리와 다른 속성을 가질 수 있으며,
    이에 맞게 테이블을 설계할 수 있다.
  
2. 새로운 카테고리가 추가될 때 기존 스키마에 영향을 미치지 않고 새로운 테이블을 추가할 수 있다.
    따라서 시스템의 확장성이 높아질 수 있다 
3. 테이블은 해당 카테고리의 고유 속성만을 가지고 있어 데이터 관리가 단순화될 수 있습니다

단점
1. 다수의 테이블을 조인해야 하는 경우 SQL 쿼리가 복잡해진다.
2. 여러 개의 테이블을 관리해야 하므로 유지보수가 번거로울 수 있다.

이 부분을 Trade Off 하여 장점이 단점에 비해 크다 생각하여 적용
```
